## OpenClaw Security Audit 🔴 CRITICAL

**Version:** 2026.3.8 (latest, stable channel)

### Findings:
- **🔴 CRITICAL:** Open group policy with elevated tools enabled
  - Slack channel has `groupPolicy="open"` while elevated tools are enabled
  - Risk: Prompt injection in Slack rooms can become high-impact incidents
  - **Recommendation:** Set `groupPolicy="allowlist"` and keep elevated allowlists extremely tight

- **🔴 CRITICAL:** Open group policy with runtime/filesystem tools exposed
  - Open Slack groups have access to exec, process, read, write, edit tools
  - sandbox=off, fs.workspaceOnly=false in agent defaults
  - Risk: Prompt injection can trigger command/file actions
  - **Recommendation:** Use `tools.profile="messaging"` for open groups, enable `tools.fs.workspaceOnly=true`, set `agents.defaults.sandbox.mode="all"`

- **🟡 WARNING:** Multi-user setup with personal-assistant model
  - Configuration suggests potential multi-user access
  - Current trust model assumes single trusted operator
  - **Recommendation:** Consider separate gateways for untrusted users
